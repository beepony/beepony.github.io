    • [Beta-Talk-02]崇尚简单的乐观派

    Beta-Talk 是我计划的一个内部各部门优秀员工访谈计划，因在贝塔咖啡录制，故起名 Beta-Talk。因各种原因，只录了两期，此为第二期，跟应用开发主管的对谈

    访谈音频

    妹子带我去战斗

    数学专业毕业，阴差阳错，被妹子和一本《疯狂的程序员》引入计算机行业，开始了程序员之路。毕业之后去上海 PPS 工作。杭州这边同学和朋友多一些，所以回到了杭州。本来想去 SF 的，对 SF 情有独钟，而且是深度用户~收到 HR 的召唤，来到了 UPYUN。

    日常工作流程

    底层研发准备好的功能，需要应用开发对接的一些 API，完成之后，前端实现客户反馈或者其他产生的需求 -> 产品 -> 应用开发

    开发眼中产品

    好产品：让开发做更少的事情，能否把原型修改迭代的更完整一些，然后再交给开发去实现
    坏产品：不擅于、不主动沟通、抓不住重点，在原型设计上花的时间少

    程序员界的语言之争

    语言不分好快，关键还是看使用语言的人。所谓的争论只是好玩而已。掌握多门语言，会不自觉的会跟自己熟悉的对比。做技术就要不断的学习，迭代，才能跟上时代的潮流，很难一门技术啃到老。

    初涉管理

    开发要遵循团队的规则，大家一起协作。比如变量命名要符合彼，追求简单清楚的代码，不适用一些奇怪难懂的。崇尚简单易懂的代码风格。

    工作中需要改进的点

    产品原型不断的迭代、改进。能够自己对原型做比较好的优化，跟开发不断调试修改代码一样。在产品设计的层面加大时间、精力投入。设计多花一两个小时，开发可以节省一两天。

    如何应对压力

    压力非常大，经常加班，既要开发新项目，又要维护老项目。性格比较乐天派，让自己成长的也很快。压力大的时候，就想回家睡觉。熬过来，就成长起来了

    今年规划

    Franky API 接口能否开放给所有的用户，让用户更加方便管理自己后台，提升平台服务能力。

    时间管理经验

    早上早点到公司，把自己的事情先安排分配一下，趁早上比较清醒、干扰比较少的时候，抓紧时间处理完毕。

    从开发到管理的角色转变

    单纯做开发时候，比较被动，单纯执行就可以了。分配任务，考虑未来的开发方向。多放权，相信自己的伙伴，自己主要做一些辅助或者指导。

    技术人的声生命周期

    十大吃青春饭的职业第二名——程序员，目前还有没有考虑，现在就是想好好的磨练技术。终极目标就是：财务自由，自由之后，希望在家里陪小孩。不喜欢加班，希望在正常的工作时间内，把该做的事情做好。认认真真写四个小时的代码，就非常好了。

    如何看待 Remote

    远程工作非常考验自己控制力和团队协作，最大的问题额外的干扰和沟通成本比较高，对团队成员的素质要求比较高。当面沟通会激发创造性。
